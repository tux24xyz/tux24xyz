Linux 發行版是什麼？

發行版（Distribution, 簡稱 Distro )，是一種在 Linux 才有的概念。要先了解的是，我們平常所說的 Linux 雖然常被認為是一種「作業系統」，但它其實只是眾多「作業系統」的「核心」，Linux 的發行版其實還比較接近「作業系統」的概念。Linux 發行版其實就是「核心」之上（以外）的程式的總和。用簡單的話來說，Linux 和 Windows or Mac 不是同一個層級（作業系統層級）的，你可以把 Linux 視為眾多「作業系統」的總稱，所以你在比較使用體驗的時候應該要拿發行版和 Windows or Mac 比，畢竟不同的發行版之間有很大的差異（這個我等一下會提到）。

注意到我剛才放了很多「」嗎？如果你不懂那些詞的話可以看我之後的解釋。

給看不懂第一段的人：

作業系統：你的電腦中比較接近底層（硬體）的軟體，可大可小。可以像 Windows or Mac 那麼肥又難搞，也可以像 Arch Linux（一個我等一下會提到的發行版）如此精簡又可調。

核心：整個作業系統中最底層（接近硬體）的部份。

一個發行版包含了什麼？

桌面環境

Desktop Environment. 你可以把它理解成你打開電腦後所看到的（幾乎）一切。裡面有「工作列」( Dock ) 、「檔案總管」( Finder ) 、「開始按鈕」以及你應該沒看過的終端機等等。對，你在 Windows/Mac 熟悉的那些東西都會出現，不過通常功能更為強大。除此之外，設定（長的像齒輪的東西）是使用桌面環境才有的東西，這點和 Windows/Mac 不一樣，我很難想像沒有這東西一般使用者要怎麼使用電腦。

視窗管理器

Window Manager. 其實如果你使用桌面環境 (以下簡稱 DE)，視窗管理器 (以下簡稱 WM) 已經包含在內了。但其實想使用電腦，只要有視窗管理器和一些輔助小工具（像是程式啟動器）就夠了。另外，DE 所使用的 WM 通常是浮動式 (Floating) 的，而單獨使用的 WM 通常是平鋪式 (Tiling) 的。我們講到 WM 通常指的是後者。

使用 WM 的人有三種可能，一種是極簡主義者、一種是追求只用鍵盤提昇效率的人、還有一種人只是想裝酷。因為 WM 的使用難度通常比 DE 高很多。這篇文章不會推薦 WM 給初學使用者。

如何安裝軟體

在開始前要先補充上一段的內容，也就是我們等一下會提到的套件管理員 (Package Manager) 。

在 Windows 下安裝軟體必須下載 .exe或 .msi 的檔案然後點開安裝，你還有可能因為跑到錯誤的網站下載到病毒。Mac 大家應該都知道了。而在 Linux，有三種方法：

  1. 用套件管理員，一種會幫你從套件庫下載套件然後幫你裝到系統上的工具。不同的發行版會有不同的套件管理員，所以在這裡就要提到三大系了：

    1. Debian 系 套件管理員和系統如何更新有很大的關係（畢竟更新就是裝新軟體嘛），Debian 系使用的套件管理員叫做 dpkg，前端叫做 apt。（前端後端不用管，因為你基本上用不到後端）。採用的是版本號的更新方式，也就是 Windows 採用的方法。以 Debian 來說，它的更新速度真的非常慢，很適合開發者做出其他的發行版，所以 Debian 系目前是 Linux 作業系統中最大的一支。
    2. Arch 系 Arch 系採滾動更新（Rolling release），意思是說系統永遠保持在最新狀態，如果太久不更新可能會造成無法挽回的後果。我最喜歡 Arch 的套件管理員有兩個原因，一個是滾動更新能讓我用到最新的軟體，不過有時候會有點小 Bug 就是了；一個是 Arch 有最大的（非官方）套件庫：AUR，詳細請看這篇文章。
    3. Fedora 系 Fedora 系採半滾動更新，意思是只有非核心軟體採用滾動更新，核心軟體還是用版本號的方式。Fedora 系使用的套件管理員為 rpm，前端叫做 dnf。Fedora 更新速度也比較快，相較於 Debian 的兩年一次大版本，Fedora 九個月就大更新一次。

  2. 手動從原始碼編譯安裝

    這個一般人不用知道。

  3. 用跨發行版的方法(Flatpak, Snap & Appimage)

    三者使用的都是把整個程式打包的方法，解決不同發行版版本不同步的問題，還能順便防止程式把你的電腦搞壞。我自己只用 Flatpak，而且我能用套件管理員就用套件管理員。我只會在 Debian/Fedora 系發行版套件庫沒有我想要的東西的時候用 Flatpak。我覺得以新手來說，只用 Flatpak 不見得是好事。我雖然同意 Luke Smith 的觀點，但用了也不會怎樣啦，有些桌面軟體我也是用 Flatpak 裝的，畢竟新手肯定解決不了依賴地獄。

首先是難度等級 - 最低： 如果你很少用電腦，不常用電腦，平常都用手機、自認是「3C 白痴」、剛才測驗只會前兩項或更少。那我會直接建議你去用 Linux Mint 。

Linux Mint 才是最適合初學者的發行版，沒有別的。因為它有內建中文輸入法、簡易的安裝過程、方便的軟體商店、各種完善的支援。而且預設界面（Cinnamon DE 版）長的像 Windows XP （不喜歡的話也可以改得很漂亮，但我想你們大多數都不在意，因為我還沒看過比 Windows 10 更醜的東西）。給一句評語：“It just works.”

如果你是在這個難度層級，但是想要比較接近 Mac OS 的使用體驗。那你也可以試試看Pop!_OS, Nobara GNOME, Fedora。不過這些我都沒用過，所以不會第一個推薦。其中如果你有玩遊戲的需求，建議用 Nobara。

順帶一提，身為 Mac 使用者的官大為老師，曾經推薦 Pop!_OS … 不過那是 2021 年 9 月的事了。

難度等級 - 中

如果你們中的有些人在 Windows/Mac 使用者中算是會一些進階使用技巧、會用電腦做一些專業的事、是「Office 大師」、「Adobe 大師」、會看那些藍藥丸科技頻道。那恭喜你來到這個級別。

如果你對自己不太有自信，請回到上一級，不要勉強，我很怕失去你們的注意力。

這一級和上一級的差別是我選了自訂性更強的 KDE（預設界面接近 Windows 11，可以改的完全不一樣） 來作為 DE。我首推 Nobara KDE，我用過，非常適合新手和玩家使用。如果你完全不想玩遊戲，也可以選擇 Fedora Spin KDE 或 Kubuntu。

新增一個：Zorin OS。這個發行版比較特殊，雖然用的是 GNOME 桌面環境，但被它們團隊魔改的超像 Windows。如果你是完全不想自訂的人，想要最接近 Windows 的使用體驗，請選 Zorin OS。

一樣，如果你想要比較接近 Mac OS 的使用體驗，我還是推薦上一級的選項。

難度等級 - 硬體玩家

如果你會自己組電腦，那你可以直接來這一級了。你很有可能會自己裝 Windows 到電腦上，所以圖形化安裝肯定難不倒你。嗯…那還有什麼會成為你的關卡呢？

那可能是終端機吧。我推薦想挑戰自己的人試試看 Endeavour OS KDE 或 ArcoLinuxB KDE。不過不要第一次就試，建議先從前面提過的發行版先開始，用 KDE 桌面環境你會比較熟悉預設介面。到這個等級，有興趣的人可以開始在 YT 上找一些 KDE 美化的教學影片並照著做，畢竟界面好看也是 Linux 的最大優點之一。如果過程中遇到問題也可以寫信來問我，我會很樂意回信幫忙的。

難度等級 - Mac

如果你是個算是會用 Mac 的使用者，那我推薦使用 GNOME 桌面環境的發行版。

同樣的桌面環境，只要在自由度夠高的發行版中就有許多選擇。

以三大系來說 Debian：Pop!_OS（它的桌面環境已經不是 GNOME 了，但也是 GNOME 的改版）、Zorin OS（如果你擅長自定義桌面，我不太推薦這個給想要舒服使用的人） Arch（先試過其他系的發行版再說）：Endeavour OS GNOME、ArcoLinuxB GNOME Fedora：Fedora、Nobara GNOME（如果你想玩遊戲）

難度等級 - 不重要

如果你已經不在乎難度，一開始的小測驗符合 6＆7 項。那你可以試著挑戰自己，使用 Endeavour OS 或 ArcoLinux，這些發行版不會有最好的支援，但我覺得是鍛鍊除錯、動手能力，前往下一階段的好地方。

等等…你是不是漏了什麼…

是的，從文章開始到現在，「Ubuntu」這個字我一次都沒提過。原因是它的母公司 Cannonical 強推自己出品的 Snap（我在安裝軟體方式 3 有提到），在 Ubuntu 下使用 apt 命令安裝軟體預設會被強制改成 Snap 安裝，Ubuntu 自己的軟體中心（類似 App Store）也不例外。而且 Snap 的伺服器不夠多，軟體又是閉源的，種種原因讓我很反感。

還有 Manjaro，我人生第一個接觸的發行版。安裝軟體方式 3 和一開始 Tierlist 的影片有提到 Manjaro 的問題。還有這個、這個、跟這個。我想也不用多講了，哪有使用 Arch 系發行版卻不能用 AUR 的道理。

Debian 和 Arch 不在這單純只是不適合新手。就像一開始講的，如果你是對技術有興趣，不怕困難的人應該使用來自最源頭的發行版，並享受自己動手、自由度超高的樂趣（這在 Windows/Mac 絕對體驗不到）。

我自己用的發行版

我現在打這篇文章是在裝了 Arch + LARBS 的破筆電上，我不推薦使用你不完全知道它做了什麼事的一鍵安裝腳本。用腳本沒問題，但你要知道它做了什麼。我是先裝了才開始學，所以也花了超多的時間除錯（我的 Nvidia 獨顯到現在還不能用，玩遊戲請找 Nobara）。我玩遊戲的電腦目前裝的是 Nobara KDE。我用過 Linux Mint Cinnamon、Manjaro KDE、Artix + i3 WM 等等發行版。還沒用過 GNOME，之後打算試試看 ArcoLinuxB GNOME。
